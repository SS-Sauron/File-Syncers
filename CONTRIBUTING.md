# Contributing to File-Syncers

Thanks for your interest in contributing! Because this tool handles real file operations, safety is our priority.

## Testing
- Always run with `--dry-run` first before committing any code that modifies files.
- Use a test directory with dummy files for development.

## Code Style
- Follow PEP 8.
- Keep the zero-dependency philosophy: use only Python standard library modules.

## Pull Request Process
1. Open an issue describing your proposed change.
2. Fork the repo and create a feature branch.
3. Test thoroughly with dry-run.
4. Submit a PR with a clear description.

Thank you for making File-Syncers better!
